# GLMV Image Analyzer

基于智谱的视觉模型对图片进行分析和理解.

## 脚本目录

**重要**：所有脚本均位于本技能的 `scripts/` 子目录中。

**Agent 执行说明**：

1. 确定本 `SKILL.md` 文件所在目录路径，记为 `{baseDir}`
2. 脚本路径 = `{baseDir}/scripts/<script_name>.ts`
3. 将本文档中所有 `{baseDir}` 替换为实际路径
4. 解析 `${BUN_X}` 运行时：若已安装 `bun` → 使用 `bun`；若有 `npx` → 使用 `npx -y bun`；否则提示用户安装 `bun`

**脚本说明**：
| 脚本 | 用途 |
|------|------|
| `scripts/main.ts` | 基于智谱视觉模型的图片分析 CLI，封装了 7 种专用分析工具，供 skill 脚本化调用。 |

## 使用方式

```bash
${BUN_X} {baseDir}/scripts/main.ts <tool> [options]
```

**查看帮助**: `${BUN_X} {baseDir}/scripts/main.ts --help`

## 可用工具

该脚本提供针对不同图形任务的专业工具:

1. **`ui_to_artifact`** - 将 UI 截图转换为各种产物
   - 从设计稿生成前端代码
   - 创建用于 UI 重建的 AI 提示词
   - 提取设计规范文档
   - 生成 UI 的自然语言描述

2. **`ocr`** - OCR 文字提取
   - 从截图中提取代码（保持正确格式）
   - 提取终端输出和日志
   - 提取文档和文本内容
   - 支持编程语言提示以提高准确性

3. **`error`** - 错误诊断和故障排查
   - 分析错误消息和堆栈跟踪
   - 识别根本原因
   - 提供可操作的解决方案
   - 建议预防策略

4. **`diagram`** - 技术图表分析
   - 分析架构图
   - 理解流程图和 UML 图
   - 解释 ER 图和序列图
   - 识别设计模式

5. **`data-viz`** - 数据可视化洞察
   - 从图表中提取洞察
   - 识别趋势和模式
   - 检测异常值
   - 提供业务影响分析

6. **`ui_diff`** - UI 对比检查（视觉回归测试）
   - 比较预期和实际的 UI 实现
   - 识别视觉差异
   - 提供详细的差异报告
   - 按严重程度排列问题优先级

7. **`analyze`** - 通用图像分析（兜底工具）
   - 当专业工具不适用时使用
   - 灵活理解任何视觉内容
   - 全面的图像描述和分析

## 参数说明

- `image_source` 支持本地文件路径（绝对/相对）和 HTTP/HTTPS URL
- 支持的图片格式：`.jpg`、`.jpeg`、`.png`，最大 5MB

### 1. `ui_to_artifact`

| 参数             | 必填 | 说明                                                                                                   |
| ---------------- | ---- | ------------------------------------------------------------------------------------------------------ |
| `--image_source` | 是   | 图片路径或 URL                                                                                         |
| `--output_type`  | 是   | 产物类型：`code`（前端代码）/ `prompt`（AI 提示词）/ `spec`（设计规范）/ `description`（自然语言描述） |
| `--user_prompt`  | 是   | 用户的具体需求                                                                                         |

### 2. `ocr`

| 参数                     | 必填 | 说明                                                    |
| ------------------------ | ---- | ------------------------------------------------------- |
| `--image_source`         | 是   | 图片路径或 URL                                          |
| `--user_prompt`          | 是   | 用户的具体需求                                          |
| `--programming_language` | 否   | 编程语言提示（如 `python`、`rust`），提高代码识别准确率 |

### 3. `error`

| 参数             | 必填 | 说明                                                 |
| ---------------- | ---- | ---------------------------------------------------- |
| `--image_source` | 是   | 图片路径或 URL                                       |
| `--user_prompt`  | 是   | 用户的具体需求                                       |
| `--context`      | 否   | 错误发生的上下文描述（如 `when running unit tests`） |

### 4. `diagram`

| 参数             | 必填 | 说明                                                             |
| ---------------- | ---- | ---------------------------------------------------------------- |
| `--image_source` | 是   | 图片路径或 URL                                                   |
| `--user_prompt`  | 是   | 用户的具体需求                                                   |
| `--diagram_type` | 否   | 图表类型提示（如 `sequence`、`er`、`flowchart`、`architecture`） |

### 5. `data-viz`

| 参数               | 必填 | 说明                                               |
| ------------------ | ---- | -------------------------------------------------- |
| `--image_source`   | 是   | 图片路径或 URL                                     |
| `--user_prompt`    | 是   | 用户的具体需求                                     |
| `--analysis_focus` | 否   | 分析侧重点（如 `trends`、`anomalies`、`patterns`） |

### 6. `ui-diff`

| 参数                      | 必填 | 说明                    |
| ------------------------- | ---- | ----------------------- |
| `--expected_image_source` | 是   | 预期/参考图的路径或 URL |
| `--actual_image_source`   | 是   | 实际/当前图的路径或 URL |
| `--user_prompt`           | 是   | 用户的具体需求          |

### 7. `analyze`

| 参数             | 必填 | 说明           |
| ---------------- | ---- | -------------- |
| `--image_source` | 是   | 图片路径或 URL |
| `--user_prompt`  | 是   | 用户的具体需求 |

## 输出格式

成功时通过 stdout 输出 JSON：

```json
{ "success": true, "data": "分析结果文本" }
```

失败时通过 stderr 输出 JSON，退出码为 1：

```json
{ "success": false, "error": "错误信息" }
```
